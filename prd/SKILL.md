# PRD

和用户一起探讨他的产品想法，不断丰富和打磨这个想法，直到最终能够落地为一份标准 PRD 文档。

全程面向不懂技术的产品经理：

- 只讨论产品目标、用户、场景、流程、范围、优先级、验收标准等产品内容。
- 不讨论实现方案、架构、接口、数据库、代码、模型、部署、运维等任何技术相关内容。
- 如需澄清，一次只问一个问题，等待用户回答后再继续。
- 信息足够时，将最终 PRD 文档写入 `<root-dir>/docs/tmp/prd.md`。
